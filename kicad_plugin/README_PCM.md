# Publishing to the KiCad Plugin & Content Manager (PCM)

The PCM is the highest-leverage distribution channel: it puts this plugin **inside the KiCad the user already has open**. This guide covers packaging and publishing.

## 1. Prerequisites

- A **64×64 PNG** icon at `kicad_plugin/icon.png` (PCM requires it). You can generate one from `assets/app_icon.ico` / `assets/icon_generator.py`.
- The plugin version in [`metadata.json`](metadata.json) matches the release you're cutting (`versions[0].version`).

## 2. Build the package

```bash
python scripts/build_pcm_package.py \
  --download-url https://github.com/PedroWall-e/EDA-Footprint-Generator/releases/download/v2.0.0/DataFrontier-PCM-2.0.0.zip
```

This produces `build/pcm/DataFrontier-PCM-<version>.zip` with the PCM layout:

```
metadata.json
plugins/          # the pcbnew ActionPlugin code
resources/icon.png
```

…and prints the `sha256`, sizes, and a ready-to-paste `versions[]` block.

## 3. Publish

### Option B — Self-hosted repository (LIVE ✅)

Already set up. The index is served via GitHub Pages from `docs/pcm/`:

- **Repository URL (share this):**
  `https://pedrowall-e.github.io/EDA-Footprint-Generator/pcm/repository.json`

Users install it in **KiCad → Plugin and Content Manager → Manage… → + (add repository)**, paste the URL, then find "EDA Footprint Generator" in the plugin list and click Install.

The index files ([`docs/pcm/packages.json`](../docs/pcm/packages.json), [`docs/pcm/repository.json`](../docs/pcm/repository.json)) are generated by [`scripts/build_pcm_repo.py`](../scripts/build_pcm_repo.py) from the **exact zip published in the release**, so the `download_sha256` always matches what KiCad downloads.

### Option A — Official PCM repository (max reach, needs submission)

Submit a merge request to the [KiCad PCM metadata repo](https://gitlab.com/kicad/addons/metadata). Once merged, the plugin appears in **every** KiCad user's PCM by default.

1. Fork `gitlab.com/kicad/addons/metadata`.
2. Copy [`docs/pcm/official/metadata.json`](../docs/pcm/official/metadata.json) to `packages/com.datafrontier.footprint-generator/metadata.json` in your fork.
3. Open a merge request and follow their review checklist (icon 64×64 ✓, license ✓, versions with sha256 ✓).

> This is the only step that must be done by a human on GitLab — the file is already generated and ready to submit.

## 4. On every new release

Per the [documentation-discipline rule](../.agents/AGENTS.md):
1. Bump the version in `kicad_plugin/metadata.json` (and `core/version.py`), add a `CHANGELOG.md` entry.
2. `gh release create vX.Y.Z` — the [`release-pcm.yml`](../.github/workflows/release-pcm.yml) workflow builds and attaches the zip automatically.
3. Regenerate the self-hosted index from the **published** asset:
   ```bash
   gh release download vX.Y.Z --dir build/pcm/_hosted --clobber
   python scripts/build_pcm_repo.py \
     --zip build/pcm/_hosted/DataFrontier-PCM-X.Y.Z.zip \
     --download-url https://github.com/PedroWall-e/EDA-Footprint-Generator/releases/download/vX.Y.Z/DataFrontier-PCM-X.Y.Z.zip \
     --pages-base https://pedrowall-e.github.io/EDA-Footprint-Generator/pcm
   ```
4. Commit `docs/pcm/*` and push. Also refresh `docs/pcm/official/metadata.json` if submitting to the official repo.

> ⚠️ Always derive the sha256 from the **hosted** asset (step 3), not a local build — cross-platform zlib differences make local and CI zips byte-different even with identical content.
